# Codex Execution Rules

## References

Before implementation, read:

- AGENTS.md
- PROJECT_STATUS.md
- CODEX_RULES.md

Use them as source of truth.

## Scope Control

- Implement only the requested sprint scope.
- Do not add extra features.
- Do not perform speculative development.
- Do not change architecture unless explicitly requested.
- Do not read old sprint notes unless required by the current sprint.
- Inspect only files directly related to the requested sprint.
- Do not scan unrelated modules or screens.

## Token Efficiency

- Do not perform full-project analysis.
- Inspect only files related to the sprint.
- Run only targeted verification.
- Avoid project-wide test runs unless required.
- Keep implementation reports concise.
- Do not repeat rules already defined in project documents.
